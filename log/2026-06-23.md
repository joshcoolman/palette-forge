# Log — 2026-06-23

A design-and-polish day on top of the shipped "surprise me" journey: a flippable
library card, a site-wide type system, a consistent circular-action vocabulary,
and a stripped, curated home. Newest at the bottom. Each beat: what changed, a
short recap, and a **Why** when there was a real decision.

## Preview infra: PORT-aware dev server

Changed: `vite.config.ts` reads `PORT` (falls back to 3000); `.claude/launch.json`
added (`autoPort`, `pnpm exec vite dev`).

So the preview tooling can assign a free port instead of colliding with an

## Library card: text-specimen face

Changed: `library-card.tsx` gained a text-specimen rendering of the six roles —
background fill, text title/body, muted subtitle, border rule, surface callout,
accent eyebrow + action pill. No labels, no hex.

**Why:** a saved palette is more legible as "how it reads applied to a UI fragment"
than as labelled swatches — the role names already imply the application.

## Global nav + site-wide font pairing (Fraunces + Inter)

Changed: `components/nav/global-nav.tsx` (persistent home / Library / Settings +
a font-pairing dropdown, each option previewed in its own fonts); a fixed set of
5 pairings (`features/typography/pairings.ts`); `lib/type-store.ts` (one global,
localStorage-persisted pairing driving `--font-heading` / `--font-body`);
`font-loader.ts` ported from type-explorer. Default **Fraunces + Inter**,
SSR-loaded in `__root` so the app opens in it with no flash; applied app-wide via
`body` + `.pf-heading` on titles.

**Why:** color without type is half a design decision. Went global (chrome + body)
because Inter is a UI font and the whole tool reads as one designed surface — the
caveat is serif-body presets serif-ify the UI, so presets should stay sans-body.
The agent-vs-deterministic question stays open (the keyless engine output is fast
and good) — typography is a preview lens, never stored on the palette or exports.

## Flip card + code popup + corner toggle + centered type

Changed: the library card became a real flip card — clicking flips swatches ↔
text specimen (`motion` rotateY, `preserve-3d`). The `T` toggle was dropped; a

**Why:** the card is the interface — flipping is more direct than a toggle, and a
single fixed corner control reads cleaner than per-face chrome.

## Keyless re-run: real variety from the simulated engine

Re-running the deterministic engine used to stack identical fours (same seed → same
output). Now each round varies, while generation stays a pure function of its
inputs (no `Math.random`):
- New optional `variation` (round index) on the `PaletteEngine.compose` seam.
  `0` = opening (unchanged); re-runs pass 1, 2, … The store threads it through
  `rerunJourney`/`runSurprise`. Claude ignores it (the model varies on its own);
  `samples.ts` stays at 0 (samples deterministic).
- `SimulatedEngine`: `pickBaseHue(source, variation)` walks the source's anchor
  colors (image seeds get genuine variety from their other colors); a small
  id-seeded `varyRecipe` jitter (±15° hue, minor sat/light, via a sin-fract
  `noise` hash) keeps single-color seeds fresh while staying in-family ("stay
  close" — user's call).

Browser-verified on `#34596e`: four rounds, each take's accent shifts per round
yet stays in the seed's family (blues stay blue, rust stays rust); scores and the
"Recommended" badge move too. All 33 tests pass (engine tests stable thanks to the
default-0 / deterministic design); prod `pnpm build` passes.

**Decision:** chose variety over gating re-run on a key — the keyless engine may
be most of the product's value (`sim-engine-may-be-enough`), so a richer keyless
loop is the better bet. Color seeds stay close to the seed; image seeds cycle
their own colors.

## Keyless re-run: bold hue rotation for image seeds

Image re-runs were too timid — the first cut "walked the image's own extracted
colors," which collapses to a ±15° wobble when the image is hue-narrow (a
bottle-cap photo of all browns/olives re-runs to itself). Fixed: image re-runs now
rotate the whole palette by the **golden angle (~137.5°) per round**, so each
re-run lands in a clearly different hue family (accent swings most; near-neutral
grounds just re-tint). The opening round (variation 0) is unchanged — the honest
read of the image. Color seeds still stay on their seed hue (the literal "from
#xxxxxx" promise) with only the small jitter — per the earlier "stay close" call.

New `simulated-variety.test.ts` (3 tests): opening is stable; image re-runs jump
>60° from the opening and >60° each step (actually ~137°); color re-runs stay
<25° from the seed. Note: golden-angle spreading intentionally lets a later round
sweep back near the start, so we assert the *consecutive* gap, not distance from
the opening. Full suite 36 green; build passes.

**Why golden angle:** successive multiples are maximally spread, so consecutive
re-runs are always a big, even jump — fixes the "re-ran it, basically the same"
feel without random clumping.

## Color re-runs: harmonic variety (color theory keyed off the seed)

Color-seed re-runs were flat — the path froze on the seed hue (old "stay close"),
so every round looked the same while image re-runs (golden-angle rotation) had
great variety. Reversed the earlier call per review: color re-runs now rotate the
base hue through a `HARMONICS` list — monochromatic / complementary / triadic /
split-complementary / analogous, ordered to jump across the wheel. Still keyed off
the chosen color (every offset is a relationship *to* the seed; round 0 stays
mono), but with the same across-round motion images get. Image path unchanged.

Browser-verified on `#e7cda4` (warm tan): Vivid accents across rounds went
gold(seed) → blue(complement) → teal → purple → cyan — clearly different,
intentional schemes each re-run. Updated `simulated-variety.test.ts`: round 1 is
~complementary (seed+180 ±25), every color re-run now >60° off the seed (was the
old <25° "stay close" assertion). 36 tests, build green.

**Why harmonic, not golden:** for a color the user deliberately chose, landing on
named relationships (complement, triadic) reads as designed variations; the
golden angle's arbitrary steps suit images, which are already arbitrary.
