# Log — 2026-06-23

A design-and-polish day on top of the shipped "surprise me" journey: a flippable
library card, a site-wide type system, a consistent circular-action vocabulary,
and a stripped, curated home. Newest at the bottom. Each beat: what changed, a
short recap, and a **Why** when there was a real decision.

## Preview infra: PORT-aware dev server

Changed: `vite.config.ts` reads `PORT` (falls back to 3000); `.claude/launch.json`
added (`autoPort`, `pnpm exec vite dev`).

So the preview tooling can assign a free port instead of colliding with an
already-running dev server. The hardcoded `--port 3000` in the `dev` script would
otherwise override any assignment.

## Library card: text-specimen face

Changed: `library-card.tsx` gained a text-specimen rendering of the six roles —
background fill, text title/body, muted subtitle, border rule, surface callout,
accent eyebrow + action pill. No labels, no hex.

**Why:** a saved palette is more legible as "how it reads applied to a UI fragment"
than as labelled swatches — the role names already imply the application.

## Global nav + site-wide font pairing (Fraunces + Inter)

Changed: `components/nav/global-nav.tsx` (persistent home / Library / Settings +
a font-pairing dropdown, each option previewed in its own fonts); a fixed set of
5 pairings (`features/typography/pairings.ts`); `lib/type-store.ts` (one global,
localStorage-persisted pairing driving `--font-heading` / `--font-body`);
`font-loader.ts` ported from type-explorer. Default **Fraunces + Inter**,
SSR-loaded in `__root` so the app opens in it with no flash; applied app-wide via
`body` + `.pf-heading` on titles.

**Why:** color without type is half a design decision. Went global (chrome + body)
because Inter is a UI font and the whole tool reads as one designed surface — the
caveat is serif-body presets serif-ify the UI, so presets should stay sans-body.
The agent-vs-deterministic question stays open (the keyless engine output is fast
and good) — typography is a preview lens, never stored on the palette or exports.

## Flip card + code popup + corner toggle + centered type

Changed: the library card became a real flip card — clicking flips swatches ↔
text specimen (`motion` rotateY, `preserve-3d`). The `T` toggle was dropped; a
`</>` button opens the export/code popup. The light/dark toggle moved to a
fixed top-right corner (outside the flipping layer, same spot on both faces) and
went bare (no border/background) with an **adaptive** icon tone for contrast
against whichever swatch it floats on. Text face centered; swatch labels left.

**Why:** the card is the interface — flipping is more direct than a toggle, and a
single fixed corner control reads cleaner than per-face chrome.

## Circular action set + delete confirm + home cleanup

Changed: shared `components/ui/icon-button.tsx` (one circular shape; parameterized
`foreground`/`background`/optional `border`) used by light/dark, code, delete,
close, and copy. Delete is now an always-visible neutral trash circle that opens a
`delete-confirm.tsx` popup (swatch band + message). New-palette → a bright white
`+`. Export popup: circular `X` close and a circular clipboard copy (with stable
scrollbar gutter). Home stripped of its hero/tagline; dropzone is now an image
icon + "Upload Image"; the seed input became a row of **curated, named colors**
lifted from a Pantone card (near-white dropped), eyedropper last.

**Why:** one consistent, clickable button vocabulary; deletion deserves a
confirm; and a curated set of characterful seeds beats an algorithmic hue sweep
(which read as muted/greyed) — "characters, not coordinates," matching the app.

## Naming: "Palette Forge", drop the "journey" metaphor

Changed: user-facing "palette-forge" title-cased to **Palette Forge** (nav, doc
title, 404, settings); "Your journey" → "Palettes" and the loading/cleared copy
made plain. Left code identifiers (`journey-store`, `startJourney`, the
`components/journey/` folder) and the IndexedDB name untouched.

**Why:** "journey" was a good build-time metaphor, not the product's nomenclature
— it's just generating color palettes.
