## AI layer — Phase 0: BYO-key foundation (`feat/ai-layer`)

Started the v2 AI-layer epic ([`docs/epic-ai-layer.md`](../docs/epic-ai-layer.md)) on a
feature branch. Phase 0 is the load-bearing seam everything else builds on: a thin,
local, serverless key layer. With no key the app is byte-for-byte unchanged.

- `prefs-repo.ts` — persist `apiKey` + `model` (IndexedDB `settings` store); new
  `ChatModel = 'haiku' | 'sonnet'` type lives here, the lowest layer.
- `lib/settings.ts` — mirror both into the synchronous in-memory settings so the UI
  can gate on *existence* without an async hop (no flash of AI UI).
- `features/agent/client.ts` — the single seam: `hasKey()` (sync gate) and
  `callAnthropic()` (streaming generator) + `collect()`. The SDK (`@anthropic-ai/sdk`)
  is `await import()`-ed on first call only, so a no-key visitor never downloads it.
  Browser-direct via `dangerouslyAllowBrowser`.
- Settings UI — new "AI touches" section (`ai-access.tsx` container + `key-entry.tsx`
  masked field with show/hide + honest storage note + `model-control.tsx`). Model
  picker only renders once a key is present.

**Why the SDK over hand-rolled fetch (reversal of my first instinct):** the epic ends
at streaming (Phase 3) and structured tool-use + multi-turn (Phase 4) — exactly where
the SDK earns its keep (typed content blocks, partial-JSON tool accumulation). The only
objection was bundle weight on a static app, and a lazy import behind `hasKey()`
dissolves it: no-key users pay zero bytes (client.ts has no consumer yet, so it's
tree-shaken entirely; once Phase 1 consumes it the dynamic import keeps it split).

Gate green: `tsc`, `eslint`, `pnpm build`. Browser-verified Settings renders the AI
section with no key and the honest note; home page untouched.

## AI layer — Phase 1: Rename (manual + AI suggestions) (`feat/ai-layer`)

Phase 1 landed as a **rename dialog**, not the spec's silent per-card swap. The
design grew through review into something better: rename is a base feature for
everyone, and AI is *lightly offered* inside it — never forced, never silent.

- **Rename is for everyone (pencil icon, "Rename this palette" tooltip).** Clicking
  opens a dialog that follows the delete/export convention — the palette's own
  `SwatchRow` on top. Manual edit (a text field + Save) always works and needs no
  key. This is a non-AI base feature, so the rename affordance is *not* key-gated.
- **AI suggestions are the gated add-on.** A "or let AI suggest" section appears in
  the dialog only when a key is set (no nag otherwise). "Suggest names" calls Haiku
  for 4 distinct two-word names; pick one to apply, or Regenerate. Loading and error
  states are *visible* — a failed call shows "Couldn't reach Anthropic. Check your
  API key in Settings." + Try again, instead of failing invisibly.
- `features/agent/rename.ts` — `suggestNames(palette, count=4)` calls Haiku with the
  ground/accent hexes + the `knowledge/naming.md` persona; **throws** on API error
  (so the dialog surfaces it) and logs to console. Pure, exported `parseModelName()`
  (one name) and `parseNameList()` (multi-line, strips list markers, dedupes, caps)
  are unit-tested in `rename.test.ts` (12 cases).
- **Remove-key in Settings.** A two-step "Remove key" confirm that *truly* deletes
  the IndexedDB record (`removeApiKey` → `idbDelete`, not a blank write) — verified
  the record reads back `undefined` after removal. Model picker + control vanish.
- `IconButton` gained an optional `title` for the hover tooltip (defaults to `label`).

**Why the redesign (reversal of the spec's silent swap):** a one-shot swap gave the
user no feedback and swallowed every error — a real misconfiguration was invisible.
A dialog makes the operation legible (loading/results/error) and lets the user *not*
use AI (manual edit) — "lightly expose, don't force AI."

**Reinterprets the prime directive:** manual rename is a base feature for all users,
so the no-key app is no longer strictly byte-for-byte — but the **AI** layer stays
absent without a key (no suggestions, no nag). The intent (AI is invisible until you
opt in) holds; the letter ("byte-for-byte") is loosened for one non-AI feature, by
explicit decision.

**Deviation from the plan (followed the code):** the plan named `square-card.tsx`
(presentational, no name); rename lives on the saved cards (`favorite-card`,
`compact-card`) and persists to the addressable record. **Locale deferred:** IP geo
adds a third-party dependency + privacy surface — holding for an explicit decision.

Verified: gate green (`tsc`, `pnpm build`, 12/12 tests); new/changed files lint-clean.
Browser — no key → pencil rename for all, dialog shows manual edit only (no AI nag),
manual rename persists; with key → "or let AI suggest" + Suggest names; bad key →
visible error + Try again (the silent-failure bug, fixed); Settings remove-key → IDB
record gone.
