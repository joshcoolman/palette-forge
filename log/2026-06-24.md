# Log — 2026-06-24

A mobile-polish and saved-grid day: made the working area comfortable on phones,
fixed a couple of small chrome details, then added a Cards/List display style for
the saved grid. Newest at the bottom. Each beat: what changed, a short recap, and
a **Why** when there was a real decision.

## README: live site link (`e186103`)

Added `Live at colorfordays.com` under the H1. The repo stays named `palette-forge`
(internal codename); the public brand is "Color for Days". No rename — repo name is
private plumbing; the hosted URL and in-app brand are independent of it.

## Mobile polish: strips, heart, header buttons (`32db9df`)

Changed: `scene-variations.tsx` (1-across strips on mobile, 3 at `sm:`; render only
the first two strips per round on mobile via an nth-child limiter — engine still
composes the full round, so data stays whole; extracted a shared `RoundStrips`),
`palette-card.tsx` (heart 20px mobile / 16px desktop), `index.tsx` (mobile-only
`flex-row-reverse` + wider gap on the working-area header).

**Why:** on a phone the round card crammed three 7-cell strips into ~310px (tiny
swatches, pinched heart), and Re-run sat 8px from the X — easy to fat-finger a
close. Putting Re-run in the screen-edge corner makes the edge a backstop against
overshoot; the destructive X moves inward. Desktop untouched throughout.

## Font dropdown: primary-only trigger (`520b2cd`)

Changed: `global-nav.tsx` — the closed pairing trigger shows just the heading font
(`current.heading ?? 'System'`) instead of the full "X + Y" label, which ran too
long. The open menu still spells out the body font beneath each option.

## Saved grid: Cards/List display style (`bfd4557`)

Changed: new `compact-card.tsx` + `view-mode-toggle.tsx`; `prefs-repo.ts`,
`settings.ts`, `index.tsx`, `preferences.tsx`, `export-modal.tsx`.

A second display style for the saved grid, for users who just want to scan colors.
New persisted `savedView` pref (cards | list, default cards), set from a header
toggle opposite the "+" and mirrored in Settings ("Palette display style"). List
cards render just the `SwatchRow` strip — no label, no flip, no specimen — with
two click behaviours by viewport:

- **Mobile**: tap a row to reveal inline controls (trash · code · light/dark) in a
  single-open accordion (parent-owned open state; tapping another row collapses the
  prior).
- **Desktop** (`>=sm` via `matchMedia`): a click jumps straight to the code popup —
  find-and-share, no inline controls.

`ExportModal` gained a light/dark preview toggle on the format-pills row (far right,
opposite Hex/CSS/Tailwind); it flips the swatch preview while the exported code
still carries both modes.

**Why:** desktop's wall-of-palettes is the fun; on a phone a lighter slice reads
better. But a pure gallery added little on desktop — giving a desktop click the
"open the code to share" job is what makes List useful there. Mobile and desktop
deliberately diverge in click behaviour, not in data. Browser-verified both
viewports; prod `vite build` green throughout.
