## Show the full palette rationale instead of truncating (299212a)

The model's per-palette rationale (rendered as `character`) was clipped to a
single line by `truncate` in both places it appears as text — the journey
results (`src/components/journey/scene-variations.tsx`) and saved favorites
(`src/components/favorites/favorite-card.tsx`). As the rationales got longer and
more descriptive, they were being cut off. Dropped `truncate` (now
`leading-snug`) so the full sentence wraps, and removed the now-redundant title
tooltip on the favorite card. The surprise-grid color strips
(`palette-card.tsx`) still keep the rationale to a hover title — that's a
deliberate "keep the color wall clean" choice, so left untouched.

## Sharpen interior-designer rationales: specific, not formulaic (43a47f4)

Eval-driven (interior-designer role, Haiku, the girls'-prom brief). We had first
pushed the persona to write *longer* rationales; the longer output then read
generic — interchangeable mood copy, a repeated "like a scene…" frame, and a
reflexive three-adjective opener, so any line could swap onto any palette
unnoticed. Rewrote the `rationale` guidance in
`knowledge/roles/interior-designer.md`: require a concrete, non-transferable
anchor (with an explicit swap test), and forbid reusing an opener / frame /
signature word across the six.

**Why relax the no-color rule:** the persona previously banned naming any color
or hue, which is part of what pushed it into pure mood copy. Now it permits a
plain-prose color/light *impression* ("gold-lit," "inky") — the thing that
actually grounds a rationale — while still banning the role-by-role hex readout
("deep greens with a cedar accent"), since the swatches already show the hexes.

**Watch-item:** the shipped default persona (`knowledge/color-theorist.md`)
authors rationales too and shows the same formulaic tendency; scoped this pass to
the dev-only interior-designer role first. (Mirrored to the default below once the
change was validated in a dev run.)

## Mirror the rationale sharpening into color-theorist (09759e6)

Validated the interior-designer change in a dev run — the rationales came back
specific and varied — so applied the same guidance to the shipped default,
`knowledge/color-theorist.md`. Kept its terse **2–4 word** tag format (didn't
lengthen it to a sentence): the change was the anti-formulaic part — a
non-transferable tag (swap test), a permitted color/light impression, the same
hex-readout ban, and no reused shape/word across the six. Both AI paths now avoid
the "cool and modern" trap.
