# Log — 2026-06-22

The whole build happened in one day. Newest at the bottom. Each beat: what
changed (+ commit), a short recap, and a one-line **Why** when there was a real
decision or reversal.

## Foundations + simulated journey (M0–M1)

Changed: `features/color` (WCAG contrast + utils), `features/palette/types`,
IndexedDB repos (`lib/db`, `palette-repo`, `key-repo`), `/knowledge/*.md` plus a
loader and frontmatter contrast-policy; the engine seam, a deterministic
`SimulatedEngine`, the `useSyncExternalStore` journey store, and the Motion
vertical descent. 25 tests.

Built the whole walkable journey with no key and no tokens.

**Why:** simulate-first — the journey's value (animation, scoring, branching, feel)
is deterministic front-end, so tune it free behind a `PaletteEngine` seam and drop
`ClaudeEngine` in later; the simulator doubles as the demo, the no-key fallback,
and the test fixture. Journey-as-descent (each move a scene you can walk back
through) over a flat grid.

Reversed: `text-on-accent: AA` is unsatisfiable (dark text on a dark accent can't
reach 4.5:1) — the verifier flagged it red on the first render; switched to
`background-on-accent`. Washed-out/samey output → tinted neutrals with a
saturation floor, wider bg↔surface gap, whole-palette hue rotation across the
four takes, color-forward cards.

## Revisitable trail + refine (M2)

Changed: store holds `rounds: VariationRound[]`; a refine bar appends a steered
round; re-picking a direction branches a fresh tail.

**Why:** rounds-as-sequence makes "scroll back, see where you came from" literal;
refine anchors to the pick so "more like this" feels responsive; branching resets
the tail (one revisitable path, not a combinatorial multi-tail UI).

Flagged: the score heuristic is hue-invariant — a "warmer" refine shows the same
score as its parent though the colors shift. Looks like a no-op; isn't.

## Claude engine behind the seam (M4)

Changed: `ClaudeEngine` implements `PaletteEngine` via a hand-written
propose → code-verify → revise loop over the Messages API; `/settings` takes a
browser-stored key; structured outputs via `output_config` + `json_schema`, no zod.
Key present → Claude, absent → simulator.

**Why:** batched fan-out (one call → four palettes, one revise call for the
failures) over N parallel chains — far cheaper on the user's key, same loop at the
batch level. Mechanism/knowledge split holds inside the engine: code computes
contrast and picks what to revise; Claude composes + writes rationale. No vision
yet (hex anchors as text).

Flagged: `border-on-surface` badge renders red though it meets its 3:1 target
(`passes` = achieved text level) — the revise loop is correct, only the badge lies.

## Design pivot: color theory → vibes → "surprise me" + the card lab · commit 9bb3a8d

Changed: `docs/plan-surprise-me.md` (the reframe) and a `/lab` scratch route
(`src/routes/lab.tsx` + `public/lab-sample.jpg`) prototyping the presentation.

The color-theory types (triadic etc.) are opaque and _invisible_ in a single-accent
UI palette — every card collapsed to "neutrals + one accent." People think in
vibes/characters (Braun, Teenage Engineering…), and the reference is usually just
an image they upload. New v1 direction: image → **four distinct UI palettes,
"surprise me,"** characters not wheel types — a simplification, not an addition.
The lab locked the card's type treatment (name bold + hex regular, one size, no
"HEX" word, muted, contrast-aware) across two treatments (proportional treemap +
palette-over-image).

**Why:** lean on the model where it's strong (reading an image's mood); the
six-role single-accent model was right all along, just mislabeled as color theory.

## Re-run appends; journeys survive refresh (IndexedDB) · uncommitted

Changed: `rerunJourney` now **appends** another fresh four instead of replacing —
prior generations stay, so you build a growing wall of rows and heart the keepers.
Narration distinguishes a re-run row ("Another four — surprise") from a refine row
("Steered…"). Journeys are now **persisted to IndexedDB** and rehydrated on
reload: added a `journeys` store (`lib/db.ts`, DB_VERSION 1→2), `JourneyState`
gained `hydrated`, the store mirrors settled state (debounced 400ms, never while a
round is `running`) and `hydrateJourney(id)` restores it. The image is deduped to
one copy — every take shares the source seed, so palettes persist with `seed.value`
stripped and re-injected from `source.value` on load. The forge route hydrates on
mount, shows a "Loading your journey…" state until ready, and `Start over` (now a
button, header + footer) explicitly clears memory **and** IndexedDB. `journey-store.test`
gained `fake-indexeddb/auto`. Gate green; browser walk confirmed re-run appends
(4→8 cards), reload restores source + all rows + heart state, and Start over wipes it.

**Why:** capitalize on the model's non-determinism — a re-run is a new row to
compare, not a replacement. And once you've started building palettes you expect
them to survive a refresh; only an explicit Start over should clear. (Diversity-
aware re-runs — telling the model what it already made — left as a deliberate later
step; sending prior output back is fiddly and its payoff unproven.)
