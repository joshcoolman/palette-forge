## Dev-only: bridge a local Anthropic key into the client (PR #8)

`vite.config.ts` now reads `.env.local`'s `ANTHROPIC_API_KEY` via `loadEnv` and
injects it into the client through a `__DEV_ANTHROPIC_KEY__` define — but **only**
for `vite dev` (`command === 'serve'`); every build emits `""`. `client.ts` gained
`resolveApiKey()` (browser-stored key wins, else the dev value behind an
`import.meta.env.DEV` guard); `hasKey()` and `callAnthropic` both route through it.
So locally the AI affordances (incl. the dev eval bar) light up from `.env.local`
with no UI key entry, while a hosted build never embeds or uses the key.

Verified: prod build passes **with and without** `.env.local` (the Vercel
checkout case); the key string is absent from `.output`; `tsc`/`eslint`/73 tests
green; and the Vercel **preview deploy succeeded** with the AI surfaces correctly
absent.

**Why:** convenience without a hosting footgun — one env var already used by
`scripts/eval.mjs` now also lights up the in-app dev tooling, gated so it can
never leak into or gate the deployed app.

## Prompt tuning: secondary ≠ accent in both AI personas

Eval-driven (interior-designer role, Haiku, the new "New Black Couch" brief): the
model was making `secondary` just a lighter/darker shade of `accent` in the *same*
hue, so the two swatches by the heart read as one color twice. Added one
principle to `knowledge/roles/interior-designer.md` — "treat secondary and accent
as two distinct colors that play off each other… don't just make the accent a
lighter or darker version of the secondary" — kept principle-based (no example
hexes, per the evals "trap"). Re-ran: secondary/accent separated into
complementary pairs across all six, cleanly. Then mirrored the identical sentence
into the shipped default `knowledge/color-theorist.md`. Also added the black-couch
brief to the `eval/prompts.md` golden set.

**Why mirror to the default:** the deterministic engine already builds `secondary`
as a hue-shifted color (`roleHexes`), so only the model path lacked the rule —
adding it aligns both paths, and keeps persona A/Bs isolating *persona*, not this
behavior. Watch-item: it slightly discourages deliberately tonal/monochrome
palettes, but a verification run showed the "surprise/bold" takes still went bold.

## Eval dev panel: show the selected brief under the bar

`EvalRunner` (`src/components/dev/eval-runner.tsx`) now renders the currently
selected prompt's brief in small muted text (`11px`, `text-white/40`) *below* the
burnt-orange panel, on the page background (a sibling outside the panel), updating
live with the dropdown. A quick read of what you're about to run while building up
the golden set. Dev-only (the panel is mounted under `import.meta.env.DEV`).

## /docs viewer: Proposals section, scrollable sidebar, archive hidden

A docs-curation pass on `src/routes/docs.tsx` (+ the first proposal):

- **Proposals section** — new `docs/proposals/` group, placed just under Plans;
  any `docs/proposals/*.md` is picked up by the existing glob. First entry:
  `docs/proposals/keyless-free-runs.md` — a captured feasibility assessment of
  free server-side AI runs (Upstash per-IP cap + global spend cap, modeled on
  `~/repos/sandbox`), marked "not scheduled." See its own page for the trade-off.
- **Sidebar scrolls on its own** — the `sticky` nav was uncapped, so lower groups
  were only reachable by scrolling the *content* to the bottom (which dragged the
  sticky sidebar up — confusing). Capped to `max-h-[calc(100vh-7rem)]` +
  `overflow-y-auto`.
- **Calmer, collapsible headers** — group titles are now same-size, non-caps,
  `--app-text` semibold with a divider, and each group folds via a full-row button
  + chevron. Build log starts collapsed (it was burying everything).
- **Fixed: the active group couldn't collapse** — the render gate had force-opened
  whichever group held the active doc. Now it's purely `collapsed`-driven, and we
  *expand* a group on navigation / `#hash` deep-link instead (so the active item
  is still revealed without being un-collapsible).
- **Archive hidden** — `docs/archive/` is filtered out of the viewer (kept in the
  repo, still linked from README/CLAUDE.md). Moved three shipped plans into it:
  `plan-ai-byo-key`, `plan-ai-prompt-to-palette`, `plan-ai-rename`.

**Why hide-don't-delete:** archived plans + the old OVERVIEW stay recoverable and
keep their inbound links intact; they just leave the visible doc list.
